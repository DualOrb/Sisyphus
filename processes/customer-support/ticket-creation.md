# Ticket Creation Guide

**Trigger:** Need to document or pass on an issue within Valley Eats.

**Check First:**
- Is there already an existing ticket for this issue?
- Which ticket type best fits (if order-related, use "order issue" so order is easily referenced)

**Steps:**
1. Click "+" in ticket tracker, select ticket type, fill required info.
2. **Required (5Ws):** What (restaurant/customer/courier), Who (names required), Where (market), Why (detail + assign call recording if helpful), When (when you spoke to them). Include contact phone number.
3. **Quality:** specific (concrete facts, not emotions), simple and to the point, self-contained (reader with zero context should understand everything).
4. **Statuses:** New = nothing done yet. Pending = working on it. Awaiting Response = waiting on customer/restaurant. Resolved = resolution reached. Closed = duplicate tickets only (write "Combining" before closing).
5. **Notes:** document every step of your process. Press Enter once only (interface allows duplicates). Notes are timestamped.
6. **Finding order number:** if ticket has no order number, click customer email > open their page > find relevant order from that date.
7. **Priority:** Immediate = unresponsive only-driver, excessive delay, restaurant confirmation. High = missing/incorrect items, wrong order, wrong address. Standard = payment issues, delivery detail changes, shift adjustments. Low = menu changes, non-urgent dev tickets.

**Escalate If:**
- Unsure which ticket type to use (ask supervisor)
- Ticket involves safety, fraud, or legal concerns
