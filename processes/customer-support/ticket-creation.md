# Ticket Creation Guide

How to create tickets properly in the Valley Eats ticket tracker.

---

## When to Create a Ticket

Create a ticket in any situation where we need to document or pass on information to someone else within Valley Eats. If there is an issue that should be addressed but you cannot handle it yourself, notate it and create a ticket so it may be reviewed and escalated if needed.

---

## How to Create a Ticket

1. Click the "+" button within the ticket tracker.
2. Specify which type of ticket you want to create.
3. Fill in all required information.

**When in doubt about ticket type:** If the issue relates to an order, create an "order issue" ticket so the order can be easily referenced. Put additional details in the "Description" field.

---

## Required Information (The 5Ws)

Every ticket must answer these five questions:

| Field | What to Include |
|-------|----------------|
| **What** | What restaurant, customer, or courier are you talking about? |
| **Who** | Who are you talking to or about? Names are required. |
| **Where** | Where are they located? Which market? |
| **Why** | Why are you making this ticket? Provide as much detail as possible. Assign the call recording if it is helpful. |
| **When** | When did you talk to them? |

Additionally, always add whichever phone number can be used to reach the person you spoke to.

---

## Ticket Quality Standards

Tickets must be:

- **Specific** — Reference concrete facts, not emotions.
- **Simple and to the point** — Do not over-explain.
- **Self-contained** — Imagine someone reading this ticket with zero prior knowledge of the issue. Would they be able to understand and interpret all the relevant information?

---

## Ticket Statuses

When creating or updating a ticket, set the status appropriately:

| Status | Meaning |
|--------|---------|
| **New** | Nothing has been done to it yet by the person it is currently assigned to. |
| **Pending** | Currently working on the ticket/issue. Lets others know someone is handling it. |
| **Awaiting Response** | We are waiting on a response from the customer or restaurant. |
| **Resolved** | A resolution has been reached with the customer and us/the restaurant, or the problem has been solved. |
| **Closed** | Only use when a customer sends in multiple tickets about the same issue adding no context, or when combining tickets. If combining, write the word "Combining" in the ticket before closing. |

---

## Adding Notes

- Type into the textbox labeled "Add Note:" and press Enter when done.
- The interface allows duplicate submissions when pressing Enter, so press only once.
- All notes have a timestamp (visible by hovering over the message).
- Notes should be specific and to the point — they are critical if the ticket needs to be passed to another dispatcher.
- Document every step of your process. Write notes in between waits (e.g., while the phone is ringing, note that you are calling the customer and why).

---

## Finding the Order Number

If a ticket comes in without an order number:

1. Click the customer's email address in the ticket.
2. This opens their customer page.
3. Look for the relevant order from that date.

---

## Categorization Rules by Ticket Type

| Situation | Ticket Type to Create |
|-----------|-----------------------|
| Customer has unknown charge or payment difficulty | Payment Issue |
| Customer missing an item or received wrong item | Missing or Incorrect Item |
| Driver not confirming orders (auto-generated) | Unresponsive Driver |
| Courier dropped their shift (auto-generated) | Drop Shift |
| Courier wants to adjust shift times | Driver Issue: Other |
| Customer received entirely wrong order | Incorrect Order |
| Customer wants to change address or instructions | Change Delivery Details |
| Restaurant has not confirmed order (auto-generated) | Restaurant Confirmation |
| Restaurant requests a callback | Restaurant Call Back |
| Order delayed 20-30 min at pickup (auto-generated) | Excessive Delay |
| Item substitution needed | Substitution |
| Technical issue needing dev team | Dev Team ticket |

---

## Priority Guidelines

- **Immediate / Urgent:** Unresponsive only driver tickets, excessive delay tickets, restaurant confirmation tickets — these directly affect active orders and must be handled as soon as possible.
- **High:** Missing/incorrect items, incorrect orders, wrong address — the customer is currently affected and waiting.
- **Standard:** Payment issues, change delivery details, driver shift adjustments, restaurant callbacks.
- **Low / Deferrable:** Menu changes, portal issues, non-urgent dev tickets — can wait for appropriate business hours or teams.
